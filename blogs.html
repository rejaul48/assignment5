FREQUENTLY ASKED QUESTIONS

What is the DOM, and how does it relate to HTML and JavaScript?

Answer:
The Document Object Model (DOM) is a programming interface that represents the structure of a document as a tree of objects. In the context of HTML, the DOM allows JavaScript to interact with and manipulate the elements and content of a webpage. Each HTML element is represented as a node in the DOM tree, enabling dynamic changes to the document's structure, style, and content through JavaScript.

How can you select multiple elements in the DOM, and what are some common methods?

Answer:
You can select multiple elements in the DOM using methods such as querySelectorAll, which returns a NodeList of all matching elements based on a specified CSS selector. For example, document.querySelectorAll('.className') selects all elements with the class className. Other methods include getElementsByClassName and getElementsByTagName, which return live HTMLCollections of matching elements.

What is event delegation, and how does it improve performance in DOM manipulation?

Answer:
Event delegation is a technique where a single event listener is added to a parent element instead of individual listeners on multiple child elements. This approach improves performance by reducing the number of event listeners in the DOM, especially in cases where elements are added or removed dynamically. When an event occurs, it bubbles up to the parent, allowing you to handle events for child elements efficiently using the parent listener.

What is the difference between innerHTML, textContent, and innerText?

Answer:
innerHTML: Allows you to get or set HTML content, including tags. Be cautious as it can expose your application to XSS attacks if not properly sanitized.

textContent: Retrieves or sets the text content of an element, excluding any HTML tags. It's faster and safer than innerHTML for text manipulation.

innerText: Similar to textContent, but it respects CSS styles and may return a different result based on the element's visibility. It can be less consistent across browsers.
